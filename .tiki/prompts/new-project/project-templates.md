# Project Templates

Templates and structures for PROJECT.md and project-config.json generation.

## PROJECT.md Template

```markdown
# {Project Name}

## Vision

{1-2 sentence vision statement describing the big picture goal}

## Goals

- {Goal 1}
- {Goal 2}
- {Goal 3}

## Target Users

{Description of primary users and their roles}

### User Needs

- {Need 1}
- {Need 2}

## Technical Constraints

- {Constraint 1 - platform, performance, security, deployment}
- {Constraint 2}

## Tech Stack

- **Language:** {language}
- **Frontend:** {framework or "N/A"}
- **Backend:** {framework or "N/A"}
- **Database:** {database or "TBD"}
- **Testing:** {test framework}

### Patterns and Conventions

- {Pattern 1 - monorepo, REST/GraphQL, folder structure}
- {Pattern 2}

## Success Criteria

- [ ] {Measurable criterion 1}
- [ ] {Measurable criterion 2}
- [ ] {Measurable criterion 3}

## Non-Goals / Out of Scope

- {Explicit exclusion 1}
- {Explicit exclusion 2}

---

*Generated by `/tiki:new-project` on {YYYY-MM-DD}*
*Edit this file directly to update project context.*
```

## project-config.json Structure

```json
{
  "createdAt": "ISO-8601 timestamp",
  "updatedAt": "ISO-8601 timestamp",
  "version": 1,
  "responses": {
    "projectName": "string",
    "vision": "string",
    "goals": ["array", "of", "goals"],
    "targetUsers": "string description",
    "userNeeds": ["array", "of", "needs"],
    "technicalConstraints": ["array", "of", "constraints"],
    "techStack": {
      "language": "string",
      "frontend": "string or null",
      "backend": "string or null",
      "database": "string or null",
      "testing": "string or null"
    },
    "patterns": ["array", "of", "patterns"],
    "successCriteria": ["array", "of", "criteria"],
    "nonGoals": ["array", "of", "exclusions"]
  }
}
```

## Section Guidelines

| Section | Purpose | Length |
|---------|---------|--------|
| Vision | Elevator pitch | 1-2 sentences |
| Goals | What to achieve | 3-5 bullets |
| Target Users | Who benefits | 2-3 sentences |
| User Needs | Problems solved | 3-5 bullets |
| Constraints | Hard requirements | As needed |
| Tech Stack | Implementation choices | Structured list |
| Success Criteria | Measurable outcomes | 3-5 checkboxes |
| Non-Goals | Explicit exclusions | 3-5 bullets |

## Minimal Template (Experiments)

For quick experiments, use a reduced format:

```markdown
# {Project Name}

## Goal

{What you're building or learning}

## Tech

- {Primary technology}

## Success

- [ ] {One clear outcome}
```
